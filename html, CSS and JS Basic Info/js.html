Browser principle


    Load Page Process:

    1. web address(domain name: www....) ->
    2. DNS server(8.8.8.8): recieve the domain name -> server convert to IP address (server:           10.10.10.10)->
    3. browswer recieve IP 10.10.10.10 -> load the page to local
    4. browser send request to domain ->
    5. server get the request of the domain with a specific port eg. www.baidu.com:80
    6. server process the required content and sent back them to browser
    7. browser recieve contents and parse them
    8. user see the page

    Multi-Process--browser:

    1. JS engine -- single thread
    2. UI rending
    3. event threads
    4. request threads
    5. timer threads

    JS

    Synchronize tasks or Asynchronous tasks
    1. First Phase: Loading default: Sunchronize task
    1. get page content
    2. html->parse: DOM tree: load contents from html tag, then head, meta, title, script. done, not body
    3. Synchronize download js file and parser ->
    4. parse and run DOM tree: load contents from html tag, then head, meta, title....body ...
    5. finish parsing
    6. rending -> download files , pics
    7. finish loading
    2.Second Phase: events process(onload) Asynchronous task

BOM and DOM

    javascript consists of ECMAScript，DOM and BOM，diff from every browser，they all have diff display styles
    • DOM is W3C's standard that all the browsers follow
    • BOM is the display style based on DOM, but all individual browser has diff display formats
    • window is an object of BOM , not js's obj
    • DOM（document object model）is API of HTML and XML
    • BOM deal with browser's window and scope，cookies that support JS would be treated as part of BOM。
