  1. if a project is too large, package will not be able to divid modules.
    Maven can divid a project to multi works easily
  2. have to cp/paste jar to WEB-INF/lib.
    Maven can store jars in storage, only need the API rather than cp the jar
  3. downloading of jars vary differently. sometimes hard to download
    Maven can download jars offically, offical jar

What is Maven

  1. Automation construction tool for Java platform
    Make->Ant->Maven->Gradle
  2. construction:
    • Java source files, framework file, JSP, HTML, pic, etc original file to produce a runnable project
    • validate->compile->test->package->verify->install->deploy
    • Maven Project Structure:

      *hello-root: project name
      *src: source files
      *pom.xml: Maven core deployment files !!run cmd.exe in the folder same with pom.xml
      *main: main code
      *test: test code
      *java: java meta files
      *resources: store framework and other tools' deployment files
  3. Maven Commands:
    • mvn clean
    • mvn compile
    • mvn test-compile: compile test codes
    • mvn test: run test
    • mvn package: pack up codes

POM

  1. Project Object Model
  2. the core deployment file of Maven project

coordinates

  1. use 3-dimension to locate a unique project:
  2. groupId: company/orginization address + project name
    <groupId>com.atguigu.maven</groupId>
  3. artifactId: module name
    <artifactId>Hello</artifactId>
  4. version
    <version>1.0.0 </version>
  5. the file path is same as com/atguigu/maven/Hello/1.0.0/Hello-1.0.0

warehouse

  1. local warehouse: maven services for local projects
  2. remote warehuose:
    • private server: maven services for LAN(local Area Network)
    • central server: maven services for all the world
    • mirror of central server: amount all the continents to share the workload of central server
  3. store:
    • maven plug-ins
    • the third party jars
    • user's own maven projects

lifecycle

  1. the order of construction can not change, must follow the order
  2. Maven core defines the abstract lifecycle, the specific tasks are completed by cookies
  3. it always construct the program from the very beginning no matter where is it running now
